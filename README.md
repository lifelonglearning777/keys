# KEYS — 基本文タイピング 開発引き継ぎ資料

中学英語の基本文(123文)を4つのモードで練習する単一ファイルWebアプリ。
DRAFT (lifelonglearning777.github.io/draft/) の姉妹アプリ。

## 公開方法
1. GitHubでリポジトリ(例: `keys`)を作成
2. `index.html` をドラッグ&ドロップでアップロード
3. Settings > Pages で公開 → `lifelonglearning777.github.io/keys/`
更新は index.html の差し替えのみ。

## 基本情報
- 完全クライアントサイド(サーバー・AI・外部API不使用)。データは localStorage キー `keys-app-v1` に保存
- 先生向け設定の初期パスコード: **7777**(設定画面から変更可。UIには表示されない)
- フォント: Space Grotesk / Space Mono / Andika / BIZ UDPGothic (Google Fonts)

## index.html の構造(検索用アンカー)
| 探すもの | 検索ワード |
|---|---|
| デザイン変数(色など) | `:root{` |
| 問題データ(123文) | `const DATA=` |
| 問題の自動コンパイル(ヒント生成等) | `DATA.forEach` |
| 採点エンジン | `function judge` / `CONTR`(短縮形) / `const SYN`(同義語) |
| 間隔反復スケジュール | `function schedule` |
| ミッション定義 | `MISSION_POOL` |
| ごほうびフレーズ(31個) | `const REWARDS=` |
| バッジ定義 | `const BADGES` |
| ホーム画面 | `function vHome` / 封印宝箱 `chestSealed` / 開封後 `RICH_CHEST` |
| 学習セッション(4モード) | `vCopy` `vStep` `vOrder` `vWrite` / 正解画面 `vDone` |
| CBT | `vCbtSetup` `vCbt` `submitCbt` |
| 記録・印刷 | `vRecords` / `vPrint`(学習のきろく) |
| 設定 | `vSettings` |

## よくある追加・変更
- **問題を足す**: `DATA` 内の該当ユニット `items` に `{ja:"...",en:"...",pat:"...",exp:"..."}` を追加(ヒント・語順カードは自動生成)。別解は `alt:["..."]`
- **別解を足す**: 該当問題の `alt:[...]` に追加(例: the U.S. は US/USA/U.S.A./America を登録済み)
- **同義語を正解にする**: `const SYN=[...]` にペアを1行追加(活用形もペアで)
- **ごほうびフレーズを足す**: `REWARDS` に `["英語","日本語"]` を追加
- **ミッションを足す**: `MISSION_POOL` に `{id,txt,chk,prog,act}` を追加(actはホームの遷移先)

## テスト
ブラウザのコンソールで `runTests()` → 採点エンジン14項目が走る。現在 14/14 合格。
デザイン変更後は: ホーム(宝箱0/1/2/3/開封後)・記録・設定・4モード・CBT・印刷 を目視確認。

## 実装済み機能の要約
4モード練習(お手本/ステップ/語順/英作文)、寛容判定+CBT厳格判定、短縮形・同義語対応、
段階ヒント(1枠コンパクト表示・カードはシャッフル)、英文の設計図(CUD配慮の色+ラベル)、
音声読み上げ(画面遷移で停止)、自信度→復習間隔調整、復習ミックス(英作文に自動2問)、
今日の宝箱(鍵穴3つ解放→本人が開封→ごほうびフレーズ収集)、バッジ29(未獲得は???)、
学習マップ(レベル別進捗バー)、CBT(セグメントUI/パレット/モーダル確認/貼り付け禁止)、
記録(折りたたみコレクション/CSV/JSON引きつぎ)、学習のきろく印刷(A4)、
効果音(WebAudio・設定でオフ可)、出題する範囲(学年)設定、Enterで次の問題、1画面レイアウト、アクセシビリティ(色なし/高コントラスト/文字サイズ/動きを減らす)。

# src版パッケージについて(このzip)

このzipは、上記の完成品に加えて**ソース部品と組み立てスクリプト**を含む開発用フルパッケージです。

## 構成
```
keys-src/
├── dist/index.html   ← 完成品(これだけでも公開できる・単体で完全)
├── src/
│   ├── shell.html    ← ページの骨組み({{STYLE}}{{SCRIPT}}に部品が入る)
│   ├── style.css     ← 全デザイン(約50KB)
│   └── app.js        ← 全ロジック+問題データ+テスト(約96KB)
├── build.js          ← 組み立てスクリプト
└── README.md         ← この資料
```

## 組み立て方(AIに任せる場合は不要)
Node.jsがある環境で `node build.js` → `dist/index.html` が生成されます。
生成物は配布中の index.html と**バイト単位で同一**であることを検証済みです。

## テスト
- 採点エンジンのテスト14本は `src/app.js` 内の `runTests()` に同梱
- 実行: 公開ページ(またはdist/index.html)をブラウザで開き、コンソールで `runTests()` → 14/14 合格が正常

## 大事なこと
- **dist/index.html 単体 = src一式** です。どちらか一方があればアプリは完全に復元できます
- 公開に必要なのは index.html だけ。src/ と build.js はGitHubに置いても置かなくても動作に影響しません
- 新しいチャットで開発を再開するとき: このzipを渡し「READMEを読んで、src/app.js の◯◯を直して、build して index.html をください」と言えばOK。部品が分かれているぶん、AIの編集ミスが起きにくくなります
